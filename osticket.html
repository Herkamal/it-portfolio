osTicket - Post-Install Configurations

This lab demonstrates the necessary changes I made to configure osTicket into a functioning ticketing system that mimics real-world IT support workflows. These configurations help simulate a professional service desk environment where roles, teams, and priorities are clearly defined.

Environments and Technologies Used

  • Microsoft Azure (Virtual Machine)
  • Remote Desktop Connection
  • osTicket

Operating System Used

  • Windows 10 Pro (21H2)

Created a custom role called Supreme Admin in Agents > Roles for full administrative access.

Created the System Administrators department in Agents > Departments for organized ticket routing.

Added a team named Online banking for interdepartmental issue handling via Agents > Teams.

Created two agents: Jane Doe and John Doe to simulate ticket handling under Agents > Add New Agent.

Added user Karen for simulating a real end-user experience.

Configured SLA levels (SEV-A, SEV-B, SEV-C) via Manage > SLA for response timing based on severity.

Set up Help Topics like Password Reset and Equipment Reset under Admin Panel > Manage > Help Topics to categorize tickets efficiently.

With all the necessary configurations completed, osTicket is now fully functional as a support ticketing system. I am able to simulate a real-world IT support process, from user submission to agent response, team collaboration, and ticket prioritization based on impact.

osTicket - Ticket Lifecycle

This lab walks through the lifecycle of support tickets within osTicket. From ticket creation to agent login, assignment, resolution, and closure, each step mimics how an actual IT support desk would handle incoming service requests.

Tools & Technologies Used

  • Microsoft Azure (Virtual Machine Hosting)
  • Remote Desktop Connection
  • osTicket (Ticketing System)
  • Windows 10 Pro (21H2)

Objective

To demonstrate how tickets are submitted by users, processed by support agents, and resolved using osTicket’s built-in tools such as SLAs, help topics, roles, and teams.

A user submits a support ticket. It enters the system's queue for agents to pick up and triage.

John logs into the Agent Panel to begin working on open tickets.

John opens and reviews Ticket #1, preparing to resolve the issue.

Jane logs into the Agent Panel to assist with support tickets.

Jane works on a business-critical banking ticket and resolves the issue.

A second user ticket is submitted under a different help topic.

John resolves Ticket #2 and marks it as closed.

A third ticket is submitted, showcasing another use case.

John resolves the final ticket in the queue, closing the full cycle.

Conclusion

Through this lab, I gained a full understanding of how tickets flow through a support system using osTicket. From multiple agent logins to help topic routing, SLAs, and resolution workflows — I was able to simulate real-world IT support desk procedures.
